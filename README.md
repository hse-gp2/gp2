# Сбор и анализ данных о книгах

Проект по сбору данных о книгах из разных источников и их анализу.

## Описание

Собрали данные о книгах из двух источников - Google Books API и Project Gutenberg. Потом объединили их, почистили и сделали анализ.

### Бизнес-задача

Нужно автоматически находить книги с неполной информацией в каталоге. Когда библиотека или магазин получает данные из разных мест, у части книг есть все данные (описание, категория, издатель, год), а у части только название и автор. Хотим научиться предсказывать какие книги надо дополнить.

**Целевая переменная:** metadata_completeness
- complete - если есть 3 или больше из этих полей: описание, категории, издатель, год
- incomplete - если меньше 3

## Данные

Всего собрано 11733 книги:
- Google Books API: 1733 записи
- Project Gutenberg: 10000 записей

Целевая переменная:
- Complete: 876 книг (7.5%)
- Incomplete: 10857 книг (92.5%)

### Структура данных

Основные поля:
- title - название книги
- author - автор
- description - описание (есть не у всех)
- categories - категории/жанры
- publisher - издатель
- year - год публикации
- page_count - количество страниц
- average_rating - рейтинг (только для Google Books)
- ratings_count - количество оценок
- language - язык книги
- source - откуда данные (google_books_api или project_gutenberg)

## Как запустить

1. Установить зависимости:
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

2. Для сбора данных через API нужен ключ от Google Books API
Создать файл `.env` в корне проекта:
```
GOOGLE_BOOKS_API_KEY=ваш_ключ_здесь
```

3. Запустить скрипты по порядку:
```bash
python src/01_api_collection.py
python src/02_scraping.py
python src/03_merge_data.py
python src/04_clean_data.py
```

4. Открыть ноутбук для анализа:
```bash
jupyter notebook src/05_eda.ipynb
```

## Основные результаты EDA

**Пропущенные значения:**
Больше всего пропусков в описаниях, рейтингах и категориях.

**Распределение данных:**
- Большинство книг на английском
- Средний объем книги: 415 страниц
- Годы публикации от очень старых до современных

**Корреляции:**
Есть связь между количеством страниц и годом публикации, между рейтингом и количеством оценок.

**Выводы:**
- Данные из разных источников сильно отличаются по полноте
- Google Books дает больше метаданных
- Project Gutenberg это в основном классика без лишних деталей
- Для предсказания полноты можно использовать источник данных, год, наличие категорий
